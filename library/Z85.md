Title: Z85 (ZeroMQ Base85)
Retrieval date: 2026-03-23
Source byte size: 12565

Alphabet: 85 printable ASCII characters (Z85 standard alphabet).
Bits per char: 32 bits encoded in 5 chars → 32/5 = 6.4 bits/char (effective).

Purpose: Efficient text encoding for binary data where zero bytes and human-readability matter; designed to map 4 bytes ↔ 5 chars exactly.

Algorithm (concise): Process input in 4-byte (32-bit) blocks. For each 32-bit value, convert to base-85 producing 5 characters using the Z85 alphabet. For inputs not divisible by 4, pad to full 4-byte block and track length or use explicit length framing.

Advantages: No padding character, exact 4-to-5 mapping simplifies buffer sizing.
